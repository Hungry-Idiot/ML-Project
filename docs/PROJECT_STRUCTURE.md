## Core Inputs

| File | Role | Notes |
| --- | --- | --- |
| `data/AMO-Bench/test.jsonl` | Main AMO-Bench data source | Do not move; local data may be ignored by Git. |
| `outputs/amo_parser_ids.txt` | Parser-gradeable AMO-P IDs | Generated by `scripts/make_amo_parser_ids.py`. |
| `outputs/amo_description_ids.txt` | Description-type AMO IDs | Excluded from parser-based evaluation. |
| `.env` | Local API credentials and model configuration | Do not commit, print, or document secrets. |
| `.env.example` | Safe environment template | Keep as documentation for required variables. |
| `requirements.txt` | Python dependency list | Includes OpenAI client, dotenv, and math verification dependency. |

## Intermediate Outputs

| File Pattern | Meaning | Regeneration |
| --- | --- | --- |
| `outputs/*.jsonl` | Per-question detailed experiment records | Regenerated by experiment scripts; may require API calls. |
| `outputs/*api_errors.jsonl` | API/network/service error logs | Operational logs; safe to regenerate but should not be treated as model failures. |
| `outputs/selector_failure_cases.*` | Filtered oracle-gap case set | Regenerated from SC3 analysis and selector outputs. |
| `outputs/math500_legacy/*` | Historical MATH-500 pilot outputs | Keep for provenance; not part of current AMO-P mainline. |

## Regenerable Files

- `outputs/amo_parser_ids.txt` and `outputs/amo_description_ids.txt` can be regenerated from `scripts/make_amo_parser_ids.py` without API calls.
- Markdown and summary outputs can usually be regenerated from their corresponding JSONL inputs if the detailed JSONL already exists.
- Raw JSONL experiment outputs require API calls and should be preserved.
- `__pycache__/` and `*.pyc` files are Python cache artifacts and should not be part of source control.

## Scripts

- Baseline and diagnostic scripts live directly under `scripts/`.
- Shared functions live under `scripts/utils/`; new experiments should reuse these helpers rather than duplicating JSONL I/O, math verification, LLM calls, clustering, or Markdown table formatting.
- Historical MATH-500 pilot scripts live under `scripts/math500_legacy/` and should remain available for provenance.
